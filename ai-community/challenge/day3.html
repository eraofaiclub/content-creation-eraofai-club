04

Using GLM-5 and Kimi K2.5 via OpenRouter

These two models are recent additions that punch way above their price class.

🧠 GLM-5 (Z.ai)
The Coding Specialist

GLM-5 dropped on February 11, 2026 and immediately made waves. It's a 744B MoE model (40B active parameters) built for complex systems design and long-horizon agent workflows. On SWE-bench Verified it scores 77.8%.

WHY YOU'D USE IT

Code, debug, backend architecture — at 1/4 the price of Claude Sonnet.

OPENROUTER SLUG
openrouter/z-ai/glm-5
CHAT COMMAND TO ADD
ADD GLM-5 VIA CHAT 
Please add GLM-5 to my available models via OpenRouter. The slug is openrouter/z-ai/glm-5 with alias "glm". It has a 205K context window.

Then switch with /model glm

MANUAL CONFIG ALTERNATIVE
JSON CONFIG 
{
  "id": "z-ai/glm-5",
  "name": "GLM-5",
  "contextWindow": 205000,
  "maxTokens": 8192
}
AGENTS.DEFAULTS.MODELS
ALIAS CONFIG 
"openrouter/z-ai/glm-5": { "alias": "glm" }
Pro tip: GLM-5 is particularly strong at iterative self-correction. For simpler coding tasks, DeepSeek V3.2 is cheaper ($0.31/M vs. $1.35/M blended).
🌊 Kimi K2.5 (Moonshot AI)
The Reasoning + Multimodal Powerhouse

Kimi K2.5 launched January 27, 2026. It's a massive 1T parameter MoE model (32B active) with native multimodal support — it can read images, generate visual code, and reason about UI screenshots.

KILLER FEATURE

Auto-caching drops repeat input costs from $0.50/M to $0.13/M — a 75% discount on cached tokens.

OPENROUTER SLUG
openrouter/moonshotai/kimi-k2.5
CHAT COMMAND TO ADD
ADD KIMI K2.5 VIA CHAT 
Please add Kimi K2.5 to my available models via OpenRouter. The slug is openrouter/moonshotai/kimi-k2.5 with alias "kimi". It has a 262K context window.

Then switch with /model kimi

MANUAL CONFIG ALTERNATIVE
JSON CONFIG 
{
  "id": "moonshotai/kimi-k2.5",
  "name": "Kimi K2.5",
  "contextWindow": 262000,
  "maxTokens": 8192
}
AGENTS.DEFAULTS.MODELS
ALIAS CONFIG 
"openrouter/moonshotai/kimi-k2.5": { "alias": "kimi" }
Pro tip: Kimi K2.5's auto-caching makes it ideal for iterative conversations where you're refining the same code or design. The longer the conversation, the more you save.
